## 📌 Why I Did This Certificate

As a **QA & Test Automation Engineer transitioning into cybersecurity**, I needed a structured, recognized foundation that covered both the theory and the tools used in real security operations. The Google Cybersecurity Certificate delivered exactly that — with hands-on work in Python, Linux, SQL, and SIEM tools like Splunk and Chronicle.

This was also my first formal exposure to the **SOC analyst workflow**: alert triage, log analysis, incident escalation, and writing incident reports. Skills I'll directly use in a Security Operations Center role — and that connect naturally to my QA background in documentation, structured testing, and anomaly detection.

| # | Course | Key Topics | Status |

| 6 | **Sound the Alarm: Detection & Response** | IDS/IPS, Suricata, log analysis, incident response lifecycle, SIEM alerts | ✅ |
| 7 | **Automate Cybersecurity Tasks with Python** | Python scripting for security, file parsing, log automation, security tools | ✅ |
| 8 | **Put It to Work: Prepare for Cybersecurity Jobs** | Incident escalation, stakeholder communication, job readiness, professional portfolio | ✅ |
| 9 | **Accelerate Your Job Search with AI** | Using AI tools to optimize job search, tailor resumes, prepare for interviews, and stand out as a candidate | ✅ |

## 🧠 Key Skills Gained

- **Threat identification** — recognizing attack types, TTPs, and indicators of compromise (IOCs)
- **Risk management** — applying NIST RMF to assess and treat organizational risk
- **SIEM operations** — writing queries, interpreting alerts, and correlating events in Splunk & Chronicle
- **Incident response** — following the full IR lifecycle: Preparation → Identification → Containment → Eradication → Recovery → Lessons Learned
- **Log analysis** — reading and filtering logs from multiple sources to identify anomalies
- **Linux for security** — navigating file systems, managing permissions, reading system logs
- **Python automation** — writing scripts to parse files, automate repetitive security tasks, and build simple tools
- **SQL for investigations** — querying databases to identify suspicious access patterns
- **Network security** — understanding protocols, traffic analysis, and network hardening techniques
- **Professional communication** — writing incident reports, escalating to stakeholders, documenting findings

## 🔗 My QA → SOC Connection

The Google certificate made the QA → SOC transition very concrete for me:

| What I did in QA | What I do now in security |
|------------------|--------------------------|
| Read test logs to find failures | Read security logs to find anomalies |
| Triage bugs by severity and priority | Triage alerts by severity and risk |
| Write bug reports with steps to reproduce | Write incident reports with evidence and timeline |
| Run regression tests to verify fixes | Verify patches didn't introduce new vulnerabilities |
| Monitor CI/CD pipelines for failures | Monitor SIEM dashboards for suspicious activity |
| Automate repetitive test tasks with scripts | Automate repetitive security tasks with Python |

> **The workflow is almost identical. The data is different.**
